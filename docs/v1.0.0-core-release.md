# v1.0.0 Core Release — Tag, Deploy, Smoke

Core platform release **without** Cloudflare Stream upload, Stripe billing, or Postmark email. Those integrations resume post-tag per [v1-release-integrations.md](v1-release-integrations.md).

## Prerequisites

All rows in **Core v1 gates** below must be ✅. See [v1.0.0-tag-checklist.md](v1.0.0-tag-checklist.md) for verification commands.

**Post-v1 optional (not tag blockers):** Stream upload, Stripe, Postmark, `CF_AIG_AUTHORIZATION`, pgvector RAG.

## 1. Tag

```bash
git fetch origin
git checkout main
git pull origin main

git tag -a v1.0.0 -m "Core production release — LMS, AI tutor, CI/CD (billing/Stream deferred)"
git push origin v1.0.0
```

## 2. Deploy from tag

```bash
gh workflow run deploy.yml --ref v1.0.0
gh run watch
```

Confirm **Post-deploy smoke test** job is green.

## 3. Smoke (workstation)

```bash
export PROD_URL=https://understandtech.app
export GITHUB_REPO=nevaquit/understandtech
export ORIGIN_IP=52.252.59.54

# Git Bash / WSL:
./scripts/smoke-test-deployment.sh
```

PowerShell equivalents:

```powershell
curl -sI https://understandtech.app/ | Select-String "HTTP|cf-ray"
curl -s https://ai.understandtech.app/health
```

## 4. Playwright regression (optional, recommended)

```bash
cd tests/e2e
cp .env.example .env   # STAGING_TEST_USER_EMAIL=e2etest, password, E2E_COURSE_PATH=/course/view.php?id=2
npx playwright test --project=chromium --workers=1
```

Expected: **8 passed**.

## 5. Post-deploy checklist

Within 30 minutes, complete [post-deployment-validation.md](post-deployment-validation.md). Skip rows marked **Deferred** (Stripe, Postmark, Stream `TEST_VIDEO_URL`).

## 6. Origin maintenance (if needed)

```bash
# Nginx vhost sync (javascript.php PHP-FPM routing)
./scripts/apply-nginx-config-remote.sh

# DB connectivity sanity (PgBouncer transaction mode)
gh workflow run recover-origin.yml
gh run watch
```

## Rollback

If STOP checks fail after deploy:

```bash
gh workflow run deploy.yml --ref <previous-tag-or-sha>
```

See [playbook §7.4](playbook.md#74-rollback-plan).
