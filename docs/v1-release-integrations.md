### Steps (Cloudflare dashboard)

1. **Stream → Upload** — upload a short test clip (e.g. 30s MP4).
2. **Stream → Settings → Signing keys** — confirm signing key PEM matches Key Vault `cf-stream-signing-key` (create key if rotating).
3. In Moodle, add a **Page** or **Lesson** activity with the Stream iframe URL pattern from [edge-serverless-orchestration skill](.cursor/skills/edge-serverless-orchestration/stream-jwt-signing.md) (JWT generated by PHP, 60s expiry).
4. Copy a fresh signed manifest URL and run:
   ```bash
   TEST_VIDEO_URL='https://customer-<id>.cloudflarestream.com/<jwt>/manifest/video.m3u8' \
     PROD_URL=https://understandtech.app ./scripts/smoke-test-deployment.sh
   ```

## Postmark (transactional email)

| Item | Status |
|------|--------|
| Moodle SMTP (`smtphosts`) | ❌ Empty on production VM (verified 2026-06-06) |
| Postmark sender signature | ❌ **User action** |
| Password-reset test email | ❌ Blocked until SMTP configured |

### Steps

1. **Postmark → Sender Signatures** — verify domain `understandtech.app` (DKIM + Return-Path).
2. Create **Server API token**; store in Key Vault (suggested name: `postmark-server-token`).
3. Moodle **Site administration → Server → Email → Outgoing mail**:
   - SMTP hosts: `smtp.postmarkapp.com:587`
   - Auth: PLAIN, username `POSTMARK_API_TOKEN`, password = server token
   - Secure: TLS
   - From: verified sender (e.g. `noreply@understandtech.app`)
4. Send test: **Server → Email → Test outgoing mail** or trigger password reset for `e2etest`.

**Automated in this session:** confirmed `smtphosts` empty via `admin/cli/cfg.php` on VM.
