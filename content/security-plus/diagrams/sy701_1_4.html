🔑 PKI Certificate Issuance Process Flow
1. Registration Authority
Receives certificate request
Validates identity evidence
→
2. Certificate Authority
Issues digital certificate
Signs with CA private key
→
3. Certificate
Binds public key to identity
Published to trust store
→
4. Key Generation
Asymmetric key pair created
Private key secured by subject
⚖️ Symmetric vs Asymmetric Encryption

🔐 Symmetric Encryption

Single shared secret key:

  • Same key encrypts and decrypts
  • Requires secure key distribution
  • Faster for bulk data at rest
  • Examples: AES-256, ChaCha20

🔓 Asymmetric Encryption

Mathematically related key pair:

  • Public key encrypts; private key decrypts
  • Enables secure key exchange in transit
  • Slower; used for signatures and key wrap
  • Examples: RSA, ECC, Diffie-Hellman

🔗 Hybrid Cryptography

Production systems combine both:

  • Asymmetric exchange establishes session key
  • Symmetric cipher protects payload data
  • TLS/SSL and S/MIME use this pattern
  • Balances security with performance
