Methods to secure data Geographic Restrictions limits access to data based on the user's physical location. Involves restricting access attempts originating from certain countries or IP addresses, aiming to prevent unauthorized access from specific regions. Often used by organizations with regulations or security concerns specific to certain geographic areas. Obfuscation Involves intentionally making data less readable or understandable. Techniques like code obfuscation or data masking can be used to obscure the original data while retaining its functionality. Helps prevent unauthorized users from easily interpreting or extracting sensitive information even if they gain access.

Methods to secure data Segmentation method involves dividing data into smaller, isolated segments. Can be achieved through data partitioning or logical separation within storage systems. Can limit the impact of a potential security breach, as only a specific segment might be compromised instead of the entire dataset. Permission Restrictions Controls access to data based on user roles and permissions. Access control systems define what users or groups can access specific data elements or functionalities. Ensures that only authorized individuals have access to sensitive information based on their designated roles and responsibilities. aka ‘Access Control ‘ e.g. RBAC on Windows
