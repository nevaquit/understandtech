Exam objective 1.1

Compare and contrast various types of security controls

Security controls are safeguards or countermeasures designed to avoid, detect, counteract, or minimize security risks to physical property, information, computer systems, or other assets. Understanding the different categories and types of security controls is fundamental to implementing a comprehensive security strategy.

Security Control Categories

Technical Controls (Logical Controls) Technical controls are implemented through technology and are designed to restrict access to systems and data. These controls are typically automated and operate without human intervention once properly configured.

Examples include:

  • Firewalls that filter network traffic based on predetermined security rules
  • Antivirus software that detects and removes malicious code
  • Intrusion Detection Systems (IDS) that monitor network traffic for suspicious activity
  • Access Control Lists (ACLs) that define which users can access specific resources
  • Encryption systems that protect data confidentiality
  • Multi-factor authentication systems that verify user identity

Managerial Controls (Administrative Controls) Managerial controls are policies, procedures, and guidelines that provide the framework for the organization's security program. These controls focus on the management of people and business processes.

Examples include:

  • Security policies that define acceptable use of IT resources
  • Risk assessment procedures that identify and evaluate threats
  • Security awareness training programs for employees
  • Incident response procedures that outline steps to take during security breaches
  • Background check requirements for personnel in sensitive positions
  • Business continuity and disaster recovery plans

Operational Controls Operational controls are implemented by people and focus on mechanisms primarily implemented and executed by people (as opposed to systems). These controls are day-to-day procedures designed to enforce security policies.

Examples include:

  • Security guards who monitor physical access to facilities
  • User account provisioning and de-provisioning procedures
  • Regular security audits and assessments
  • Backup and recovery operations
  • Change management processes
  • Separation of duties to prevent fraud

Physical Controls Physical controls are tangible measures designed to prevent or detect unauthorized access to physical areas, systems, or assets. These controls protect against environmental threats and unauthorized physical access.

Examples include:

  • Locks, deadbolts, and electronic access controls on doors
  • Security cameras and surveillance systems
  • Fencing and barriers around facilities
  • Environmental controls (fire suppression, HVAC systems)
  • Cable locks for securing equipment
  • Biometric scanners for physical access control

Security Control Types

Preventive Controls Preventive controls are proactive measures designed to stop security incidents from occurring in the first place. These controls aim to eliminate or reduce the likelihood of threats materializing.

Examples include:

  • Firewalls that block unauthorized network traffic
  • Access controls that prevent unauthorized users from accessing systems
  • Security awareness training that educates users about threats
  • Antivirus software that prevents malware infections
  • Physical barriers that prevent unauthorized access to facilities

Deterrent Controls Deterrent controls are designed to discourage potential attackers from attempting to compromise security. While they don't physically prevent attacks, they create psychological barriers that may dissuade malicious actors.

Examples include:

  • Security cameras visible to potential intruders
  • Warning signs about security monitoring
  • Security guards patrolling facilities
  • Legal consequences clearly stated in acceptable use policies
  • Audit logs that track user activities

Detective Controls Detective controls are designed to identify and alert on security incidents that have already occurred or are in progress. These controls help organizations discover breaches and respond appropriately.

Examples include:

  • Intrusion Detection Systems (IDS) that monitor for suspicious network activity
  • Security Information and Event Management (SIEM) systems that correlate security events
  • Audit logs that record user activities for later review
  • Motion detectors that sense unauthorized physical access
  • Vulnerability scanners that identify security weaknesses

Corrective Controls Corrective controls are implemented to remediate or minimize the impact of security incidents after they have been detected. These controls help restore systems to normal operations and prevent similar incidents.

Examples include:

  • Incident response procedures that guide recovery efforts
  • System backups that allow restoration of compromised data
  • Patch management processes that fix security vulnerabilities
  • Quarantine systems that isolate infected devices
  • Disaster recovery procedures that restore business operations

Compensating Controls Compensating controls are alternative measures implemented when primary controls are not feasible or cost-effective. These controls provide equivalent protection through different means.

Examples include:

  • Manual review processes when automated controls aren't available
  • Additional monitoring when encryption isn't possible
  • Segregation of duties when technical access controls are insufficient
  • Enhanced logging when real-time monitoring isn't feasible
  • Physical security measures when technical controls are inadequate

Directive Controls Directive controls provide guidance and direction to personnel regarding expected behavior and security requirements. These controls establish the framework for security-conscious behavior.

Examples include:

  • Security policies that define acceptable use of IT resources
  • Procedures that outline specific steps for security tasks
  • Standards that specify technical requirements for security controls
  • Guidelines that provide recommendations for security practices
  • Training materials that educate users about security responsibilities
