Exam objective 3.3

Compare and contrast concepts and strategies to protect data

Protecting data is fundamental to information security. This involves understanding different data types, classifications, and states, as well as various methods to secure them.

Data Types

  • Regulated: Data subject to specific laws, regulations, or industry standards (e.g., HIPAA, GDPR, PCI DSS).
  • Trade Secret: Confidential information that provides a business with a competitive edge (e.g., formulas, designs, processes).
  • Intellectual Property: Creations of the mind, such as inventions, literary and artistic works, designs, and symbols, names and images used in commerce.
  • Legal Information: Data related to legal cases, compliance, or regulatory matters.
  • Financial Information: Sensitive financial data, including credit card numbers, bank account details, and transaction records.
  • Human- and Non-human-readable: Data that can be understood by humans (e.g., text documents) versus data that requires a machine to interpret (e.g., binary files, encrypted data).
  • Public: Data that is intentionally made available to the general public and has no confidentiality requirements.
  • Restricted: Data that is not public but has a lower level of sensitivity than confidential or private data.
  • Private: Data that is highly sensitive and restricted to a specific group of individuals or for internal use only.
  • Critical: Data that is essential for the organization's operations and whose loss or compromise would have a severe impact.

Data Classifications

  • Sensitive: Data that, if disclosed, could cause harm or embarrassment to an individual or organization.
  • Confidential: Data that is highly sensitive and restricted to a specific group of individuals or for internal use only.

General Data Considerations

  • Data States:

- Data at rest: Data stored on a device (e.g., hard drive, USB drive, cloud storage). Typically protected by encryption. - Data in transit: Data moving across a network (e.g., internet, internal network). Protected by protocols like TLS, IPSec, or VPNs. - Data in use: Data currently being processed, accessed, or manipulated by a system or application (e.g., data in RAM, CPU registers). More challenging to protect.

  • Data Sovereignty: The concept that data is subject to the laws and governance structures of the country in which it is stored.
  • Geolocation: The geographical location of data storage or processing, which can have implications for data sovereignty and compliance.

Methods to Secure Data

  • Geographic restrictions: Limiting where data can be stored or accessed based on geographical location to comply with data sovereignty laws.
  • Encryption: Transforming data into a coded format to prevent unauthorized access. (See section 1.4 for details).
  • Hashing: Creating a fixed-size string of characters from data for integrity verification. (See section 1.4 for details).
  • Masking: Obscuring sensitive data with realistic but false data, often used for testing or training environments.
  • Tokenization: Replacing sensitive data with a unique, non-sensitive identifier (token) that retains all the necessary information without compromising security.
  • Obfuscation: Techniques used to make code or data difficult to understand or analyze. (See section 1.4 for details).
  • Segmentation: Isolating data into different network segments or storage locations based on sensitivity or access requirements.
  • Permission restrictions: Limiting access to data based on user roles and privileges (e.g., read-only, write access).
