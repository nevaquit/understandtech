Exam objective 4.6

Given a scenario, implement and maintain identity and access management

Identity and Access Management (IAM) is a framework of policies and technologies that enables an organization to manage digital identities and control user access to resources.

Identity and Access Management

  • Provisioning/De-provisioning User Accounts: Creating new user accounts with appropriate permissions and disabling/deleting accounts when users leave the organization or no longer require access.
  • Permission Assignments and Implications: Granting specific rights and privileges to users or groups based on their roles and responsibilities. Understanding the security implications of over-privileged accounts.
  • Identity Proofing: The process of verifying the identity of an individual to ensure they are who they claim to be, especially during account creation.
  • Federation: A system that allows users to access resources across multiple, independent domains with a single identity.
  • Single Sign-on (SSO): An authentication scheme that allows a user to log in with a single ID and password to gain access to multiple related but independent software systems.
    • Lightweight Directory Access Protocol (LDAP): A protocol for accessing and maintaining distributed directory information services.
    • Open Authorization (OAuth): An open standard for access delegation, commonly used as a way for Internet users to grant websites or applications access to their information on other websites but without giving them the passwords.
    • Security Assertions Markup Language (SAML): An XML-based open standard for exchanging authentication and authorization data between an identity provider and a service provider.
  • Interoperability: The ability of different systems and organizations to work together seamlessly to manage identities and access.
  • Attestation: The process of verifying the integrity and authenticity of a system or component.
  • Access Controls: (See section 2.5 for general access control concepts)
    • Mandatory Access Control (MAC): A security model where access rights are enforced by a central authority based on security labels assigned to subjects and objects.
    • Discretionary Access Control (DAC): A security model where the owner of a resource controls who can access it and what permissions they have.
    • Role-based Access Control (RBAC): Access permissions are based on a user's role within the organization.
    • Rule-based Access Control: Access is granted or denied based on a set of predefined rules.
    • Attribute-based Access Control (ABAC): Access decisions are based on attributes of the user, resource, and environment.
    • Time-of-day restrictions: Limiting access to resources during specific hours.
    • Least privilege: Granting only the minimum necessary permissions.
  • Multifactor Authentication (MFA): (See section 1.2 for details)
    • Implementations:
      • Biometrics: Fingerprints, facial recognition, iris scans.
      • Hard/soft authentication tokens: Physical devices or software applications that generate one-time passwords.
      • Security keys: Hardware devices that provide strong, phishing-resistant authentication.
    • Factors: Something you know, something you have, something you are, somewhere you are.
  • Password Concepts:
    • Password best practices: Length, complexity, uniqueness, regular changes.
    • Password managers: Tools to securely store and manage complex passwords.
    • Passwordless: Authentication methods that do not require a password (e.g., biometrics, FIDO2).
  • Privileged Access Management (PAM) tools: Solutions designed to manage and secure privileged accounts.
    • Just-in-time permissions: Granting elevated privileges only when needed and for a limited time.
    • Password vaulting: Securely storing and managing privileged account passwords.
    • Ephemeral credentials: Short-lived credentials that are automatically generated and destroyed after use.
