Exam objective 4.9

Given a scenario, use data sources to support an investigation

Effective security investigations rely on various data sources to gather evidence, understand the scope of an incident, and identify the root cause.

Log Data

  • Firewall Logs: Record network traffic that is allowed or denied by the firewall, providing insights into attempted intrusions or unauthorized communications.
  • Application Logs: Record events within applications, such as user logins, data access, errors, and specific application activities.
  • Endpoint Logs: Logs from individual devices (e.g., workstations, servers) that record system events, process execution, file access, and network connections.
  • OS-specific Security Logs: Operating system logs (e.g., Windows Event Logs, Linux syslog) that record security-related events like authentication attempts, privilege changes, and system errors.
  • IPS/IDS Logs: Records of detected intrusions or suspicious network activity from Intrusion Prevention Systems (IPS) and Intrusion Detection Systems (IDS).
  • Network Logs: Logs from network devices (e.g., routers, switches) that provide information about network connectivity, traffic flow, and device status.
  • Metadata: Data about data, such as file creation dates, modification times, author information, and file sizes, which can be crucial for forensic analysis.

Data Sources

  • Vulnerability Scans: Reports from vulnerability scans can identify known weaknesses that might have been exploited.
  • Automated Reports: Scheduled reports from security tools (e.g., SIEM, antivirus) that summarize security events or compliance status.
  • Dashboards: Visual representations of security data that provide real-time insights into the security posture and ongoing incidents.
  • Packet Captures: Recordings of network traffic that can be analyzed to understand the details of an attack, including protocols used, data exchanged, and attacker communications.

🕵️ Digital Forensics Investigation Process

  1. Identification: Recognize and locate potential evidence sources across all relevant systems
  2. Preservation: Protect evidence from modification using write blockers and proper procedures
  3. Collection: Gather evidence using forensically sound methods and maintain chain of custody
  4. Examination: Process collected data to extract relevant information and artifacts
  5. Analysis: Interpret findings, correlate events, and draw conclusions about incidents
  6. Presentation: Report findings in clear, actionable format for stakeholders and legal proceedings
