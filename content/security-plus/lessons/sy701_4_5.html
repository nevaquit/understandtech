Exam objective 4.5

Given a scenario, modify enterprise capabilities to enhance security

Modifying enterprise capabilities to enhance security involves implementing and configuring various security technologies and practices.

Security Enhancement Technologies

  • Firewall: A network security device that monitors and filters incoming and outgoing network traffic based on an organization's previously established security policies.
    • Rules: Specific instructions that dictate what traffic is allowed or denied.
    • Access lists: Lists of permissions used by firewalls and routers to control network traffic.
    • Ports/protocols: Firewalls can filter traffic based on port numbers and protocols (e.g., TCP 80 for HTTP).
    • Screened subnets (DMZ): A perimeter network that protects an organization's internal local area network (LAN) from untrusted traffic, usually from the internet.
  • IDS/IPS:
    • Trends: Analyzing patterns in network traffic to detect anomalies.
    • Signatures: Predefined patterns of known attacks that IDS/IPS systems use for detection.
  • Web filter: A tool that controls access to websites based on content categories, URLs, or reputation.
    • Agent-based: Web filtering software installed on individual endpoints.
    • Centralized proxy: All web traffic is routed through a central proxy server for filtering.
    • Universal Resource Locator (URL) scanning: Inspecting URLs for malicious content or phishing attempts.
    • Content categorization: Classifying websites based on their content (e.g., social media, gambling).
    • Block rules: Rules that prevent access to specific websites or categories.
    • Reputation: Blocking access to websites with a known bad reputation.
  • Operating system security:
    • Group Policy: A feature of Microsoft Windows Active Directory that controls the working environment of user accounts and computer accounts.
    • SELinux (Security-Enhanced Linux): A security module for the Linux kernel that provides a mechanism for supporting access control security policies.
  • Implementation of secure protocols:
    • Protocol selection: Choosing secure protocols (e.g., HTTPS, SSH, SFTP) over insecure ones.
    • Port selection: Using non-standard ports for services to avoid common attack vectors (security through obscurity).
    • Transport method: Ensuring secure transport of data (e.g., TLS for web traffic).
  • DNS filtering: Blocking access to malicious domains or categories of websites at the DNS level.
  • Email security:
    • Domain-based Message Authentication, Reporting, and Conformance (DMARC): An email authentication protocol that helps protect against email spoofing and phishing.
    • DomainKeys Identified Mail (DKIM): An email authentication method designed to detect email spoofing.
    • Sender Policy Framework (SPF): An email authentication method designed to detect sender address forgery.
  • File integrity monitoring (FIM): Monitoring critical system files for unauthorized changes.
  • DLP (Data Loss Prevention): Systems that prevent sensitive data from leaving the organization's control.
  • Network access control (NAC): A solution that restricts the availability of network resources to endpoint devices that comply with a defined security policy.
  • Endpoint detection and response (EDR)/extended detection and response (XDR):
    • EDR: Focuses on endpoints (laptops, servers) to detect and investigate suspicious activities.
    • XDR: Expands on EDR by integrating security data from multiple sources (endpoints, network, cloud, email) for broader threat detection and response.
  • User behavior analytics (UBA): Analyzing user activity patterns to detect anomalies that may indicate a security threat.
