Exam objective 2.2

Explain common threat vectors and attack surfaces

Threat vectors are the paths or methods that attackers use to gain access to systems or deliver malicious payloads. Attack surfaces represent the total number of possible entry points that an attacker could exploit. Understanding these concepts is crucial for implementing effective defensive measures.

Message-based Threats

Message-based threats use various communication channels to deliver malicious content or trick users into performing harmful actions.

Email Email remains one of the most common attack vectors due to its ubiquity and the trust users place in email communications.

Common email-based attacks:

  • Phishing emails that trick users into revealing credentials or personal information
  • Malware attachments that execute when opened by recipients
  • Business Email Compromise (BEC) attacks targeting financial transactions
  • Spear phishing attacks targeting specific individuals or organizations
  • Email spoofing to impersonate trusted senders

Email security measures:

  • Spam filters and anti-malware scanning
  • Sender Policy Framework (SPF) and DKIM authentication
  • Domain-based Message Authentication, Reporting, and Conformance (DMARC)
  • User education and awareness training
  • Email encryption for sensitive communications

SMS (Short Message Service) SMS-based attacks, also known as "smishing," exploit the trust users place in text messages and the limited security features of SMS.

Common SMS attacks:

  • Phishing messages with malicious links
  • Fake alerts claiming account compromises
  • Two-factor authentication bypass attempts
  • Premium rate service scams
  • Malware distribution through malicious links

SMS security considerations:

  • Limited encryption and authentication capabilities
  • Difficulty in verifying sender authenticity
  • User tendency to trust text messages
  • Limited security controls on mobile devices
  • Potential for SIM swapping attacks

Instant Messaging Instant messaging platforms can be exploited to deliver malware, conduct social engineering attacks, or steal credentials.

IM-based attack methods:

  • Malicious file transfers containing malware
  • Phishing links shared through chat messages
  • Social engineering to extract sensitive information
  • Account takeover through credential theft
  • Spreading malware through infected contacts

Other Attack Vectors

Image-based Attackers can embed malicious code within image files or use images as part of social engineering attacks.

Image-based attack techniques:

  • Steganography to hide malicious code in image files
  • Malicious image files that exploit image processing vulnerabilities
  • QR codes containing malicious URLs or commands
  • Fake images used in social engineering attacks
  • Image-based CAPTCHA bypass techniques

File-based Malicious files can be distributed through various channels to compromise systems or steal information.

Common file-based attacks:

  • Malware-infected documents (Word, Excel, PDF files)
  • Executable files disguised as legitimate software
  • Archive files containing multiple malicious components
  • Script files (PowerShell, JavaScript, VBScript) with malicious code
  • Macro-enabled documents that execute harmful commands

File security measures:

  • Antivirus and anti-malware scanning
  • File type restrictions and allow lists
  • Sandboxing for suspicious file analysis
  • Digital signatures for file authenticity verification
  • User education about safe file handling practices

Voice Call Voice-based attacks, known as "vishing" (voice phishing), use phone calls to manipulate victims into revealing sensitive information or performing harmful actions.

Vishing attack techniques:

  • Impersonating trusted organizations (banks, government agencies)
  • Creating urgency to pressure victims into quick decisions
  • Using caller ID spoofing to appear legitimate
  • Social engineering to extract personal information
  • Voice over IP (VoIP) systems for anonymous calling

Removable Device USB drives and other removable media can be used to introduce malware or steal data from air-gapped systems.

Removable device attacks:

  • USB drops containing malware-infected devices
  • BadUSB attacks that reprogram device firmware
  • Data exfiltration through unauthorized removable media
  • Keyloggers and other surveillance tools
  • Bypassing network security controls

Removable device security:

  • Device control policies and technical restrictions
  • Endpoint protection with removable media scanning
  • Data loss prevention (DLP) solutions
  • User education about removable device risks
  • Physical security controls for sensitive areas

Vulnerable Software Unpatched or misconfigured software provides entry points for attackers to exploit known vulnerabilities.

Software vulnerability exploitation:

  • Operating system vulnerabilities and missing patches
  • Application vulnerabilities in web browsers and plugins
  • Third-party software with known security flaws
  • Default configurations with weak security settings
  • Legacy software without security updates

Unsupported Systems Systems that no longer receive security updates present significant risks as new vulnerabilities cannot be patched.

Risks of unsupported systems:

  • No security patches for newly discovered vulnerabilities
  • Incompatibility with modern security tools
  • Compliance and regulatory issues
  • Increased attack surface over time
  • Potential for lateral movement to other systems

Unsecure Networks Networks with weak security controls or misconfigurations can be exploited to gain unauthorized access or intercept communications.

Network security vulnerabilities:

  • Unencrypted wireless networks
  • Weak authentication mechanisms
  • Default credentials on network devices
  • Misconfigured firewalls and access controls
  • Lack of network segmentation

Open Service Ports Unnecessary open ports provide potential entry points for attackers to access network services.

Port security considerations:

  • Regular port scanning to identify open services
  • Principle of least privilege for network access
  • Firewall rules to restrict unnecessary connections
  • Service hardening and secure configurations
  • Network monitoring for unusual port activity

Default Credentials Many devices and applications ship with default usernames and passwords that are often not changed during deployment.

Default credential risks:

  • Widely known default login combinations
  • Easy targets for automated attack tools
  • Administrative access to critical systems
  • Potential for lateral movement within networks
  • Compliance violations and audit findings

Supply Chain Supply chain attacks target the software or hardware development and distribution process to compromise end users.

Supply chain attack methods:

  • Compromising software development environments
  • Injecting malware into software updates
  • Hardware tampering during manufacturing or shipping
  • Third-party vendor compromises affecting customers
  • Open source software package compromises

Human Vectors/Social Engineering

Social engineering attacks exploit human psychology and behavior rather than technical vulnerabilities. These attacks are often highly effective because they target the weakest link in security: people.

Phishing Phishing attacks use fraudulent communications to trick victims into revealing sensitive information or performing harmful actions.

Phishing characteristics:

  • Mass distribution to many potential victims
  • Generic messages that could apply to anyone
  • Urgency or fear tactics to prompt quick action
  • Requests for sensitive information or credentials
  • Links to fake websites that mimic legitimate services

Common phishing techniques:

  • Email phishing with malicious links or attachments
  • Website spoofing to steal login credentials
  • Fake security alerts claiming account compromises
  • Prize or lottery scams requesting personal information
  • Charity scams exploiting current events or disasters

Vishing (Voice Phishing) Vishing uses phone calls to manipulate victims into revealing information or performing actions that compromise security.

Vishing tactics:

  • Impersonating bank representatives or government officials
  • Creating false emergencies requiring immediate action
  • Using personal information gathered from other sources
  • Caller ID spoofing to appear more legitimate
  • Recording calls to gather additional information

Smishing (SMS Phishing) Smishing uses text messages to conduct phishing attacks, often directing victims to malicious websites or phone numbers.

Smishing characteristics:

  • Short, urgent messages with limited context
  • Links to malicious websites or phone numbers
  • Requests to call premium rate numbers
  • Fake alerts about account problems or security issues
  • Exploitation of mobile device limitations

Misinformation/Disinformation These attacks spread false or misleading information to influence public opinion, create confusion, or undermine trust in institutions.

Misinformation vs. Disinformation:

  • Misinformation: False information spread without malicious intent
  • Disinformation: Deliberately false information spread with malicious intent

Common targets:

  • Political processes and elections
  • Public health information and policies
  • Financial markets and economic stability
  • Social issues and community relations
  • Corporate reputation and brand trust

Impersonation Attackers pretend to be trusted individuals or organizations to gain victims' confidence and extract information or access.

Impersonation techniques:

  • Pretending to be IT support personnel
  • Impersonating executives or authority figures
  • Posing as vendors or business partners
  • Claiming to be law enforcement or government officials
  • Using social media profiles to build false identities

Business Email Compromise (BEC) BEC attacks target businesses by compromising or spoofing executive email accounts to authorize fraudulent transactions.

BEC attack patterns:

  • CEO fraud targeting finance departments
  • Invoice fraud with fake vendor payment requests
  • Account compromise through credential theft
  • Attorney impersonation for urgent legal matters
  • Real estate transaction fraud

Pretexting Pretexting involves creating fabricated scenarios to engage victims and extract information or gain access to systems.

Pretexting scenarios:

  • IT support calls requesting password verification
  • Survey or research requests for personal information
  • Emergency situations requiring immediate assistance
  • Vendor or contractor verification calls
  • Background check or employment verification requests

Watering Hole Watering hole attacks compromise websites frequently visited by target organizations or individuals to deliver malware to specific victims.

Watering hole attack process: 1. Identify websites commonly visited by targets 2. Compromise the identified websites 3. Inject malicious code into the websites 4. Wait for targets to visit and become infected 5. Use initial compromise for further attacks

Brand Impersonation Attackers create fake websites, emails, or communications that mimic trusted brands to steal credentials or distribute malware.

Brand impersonation techniques:

  • Fake websites with similar domain names
  • Spoofed emails using company logos and branding
  • Fake mobile applications mimicking legitimate services
  • Social media accounts impersonating brands
  • Fake customer support communications

Typosquatting Typosquatting involves registering domain names that are similar to legitimate websites to capture traffic from users who make typing errors.

Typosquatting methods:

  • Common misspellings of popular domain names
  • Different top-level domains (.com vs .net)
  • Character substitution (replacing letters with numbers)
  • Additional or missing characters in domain names
  • Homograph attacks using similar-looking characters
📨 Attack Vector Categories - Comprehensive Threat Landscape

📧 Email-Based Attack Vectors

  • Phishing: Generic credential harvesting campaigns
  • Spear Phishing: Targeted individual attacks
  • Whaling: Executive and high-value target focus
  • Business Email Compromise (BEC): CEO fraud and wire fraud
  • Smishing: SMS-based phishing attacks
  • Vishing: Voice-based social engineering
  • Email Spoofing: Sender identity impersonation

🌐 Web-Based Attack Vectors

  • Drive-by Downloads: Automatic malware execution
  • Watering Hole Attacks: Compromised frequently visited sites
  • Typosquatting: Misspelled domain exploitation
  • Clickjacking: Hidden malicious interface elements
  • Cross-Site Scripting (XSS): Client-side script injection
  • SQL Injection: Database query manipulation
  • Session Hijacking: Authentication token theft

📱 Mobile and IoT Attack Vectors

  • Malicious Applications: Trojanized mobile apps
  • Device Hijacking: Remote device control
  • Firmware Exploitation: Low-level system compromise
  • Bluetooth Attacks: Bluejacking and bluesnarfing
  • NFC Exploitation: Near-field communication attacks
  • IoT Botnet Recruitment: Device conscription
  • SIM Swapping: Mobile account takeover

💾 Physical Attack Vectors

  • USB Drop Attacks: Malicious portable media
  • BadUSB Attacks: Firmware-level USB exploitation
  • Card Skimming: Payment card data theft
  • Tailgating: Unauthorized physical access
  • Dumpster Diving: Information gathering from waste
  • Shoulder Surfing: Visual information theft
  • Lock Picking: Physical security bypass
