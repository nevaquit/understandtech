Exam objective 2.4

Given a scenario, analyze indicators of malicious activity

Indicators of malicious activity are signs that an attack or compromise may be occurring or has occurred. Recognizing these indicators is crucial for timely detection and response.

Malware Attacks

  • Ransomware: Malware that encrypts a victim's files and demands a ransom payment, typically in cryptocurrency, for decryption. Indicators include encrypted files, ransom notes, and unusual network traffic.
  • Trojan: Malicious software disguised as legitimate software. It performs malicious actions (e.g., creating backdoors, stealing data) while appearing to perform its intended function.
  • Worm: Self-replicating malware that spreads across networks without human interaction, often consuming network bandwidth or system resources.
  • Spyware: Malware designed to secretly observe and collect information about a user's activities without their knowledge or consent (e.g., browsing history, keystrokes).
  • Bloatware: Software that comes pre-installed on devices and consumes system resources without providing significant value to the user. While not inherently malicious, it can sometimes have security vulnerabilities or privacy concerns.
  • Virus: Malicious code that attaches itself to legitimate programs or documents and spreads when the infected program is executed. Requires human interaction to spread.
  • Keylogger: Software or hardware that records every keystroke made on a computer, often used to steal passwords or sensitive information.
  • Logic Bomb: Malicious code intentionally inserted into a software system that lies dormant until a specific condition is met (e.g., a specific date, a user's termination) to trigger a malicious function.
  • Rootkit: A collection of software tools designed to enable continued privileged access to a computer while actively hiding its presence from administrators and other security software.

Physical Attacks

  • Brute Force: Repeated, systematic attempts to guess passwords, encryption keys, or other credentials, often by trying every possible combination.
  • Radio Frequency Identification (RFID) Cloning: Copying the data from an RFID tag to another tag, often used for unauthorized access to systems or facilities.
  • Environmental: Attacks or disruptions caused by environmental factors, such as extreme temperatures, humidity, or natural disasters, leading to system failures or data loss.

Network Attacks

  • Distributed Denial-of-Service (DDoS): An attack where multiple compromised computer systems attack a target, such as a server, website, or other network resource, and cause a denial of service for users of the targeted resource.
  • Domain Name System (DNS) Attacks: Attacks targeting the DNS infrastructure, such as DNS poisoning, DDoS against DNS servers, or DNS amplification attacks.
    • Amplified: A type of DDoS attack that uses open DNS resolvers to magnify the attack traffic.
    • Reflected: A type of DDoS attack that uses legitimate third-party servers to reflect attack traffic onto a target.
  • Wireless Attacks: Attacks targeting wireless networks, such as rogue access points, deauthentication attacks, or cracking Wi-Fi encryption.
  • On-path (Man-in-the-Middle) Attacks: An attacker secretly relays and possibly alters the communication between two parties who believe they are directly communicating with each other.
  • Credential Replay: Capturing legitimate authentication credentials and re-using them to gain unauthorized access.
  • Malicious Code: Code injected into network traffic or web pages to exploit vulnerabilities.

Application Attacks

  • Injection: Inserting malicious code or commands into an application's input fields (e.g., SQL injection, command injection).
  • Buffer Overflow: Exploiting a buffer overflow vulnerability to execute arbitrary code.
  • Replay: Capturing and re-transmitting valid data transmissions to gain unauthorized access or perform unauthorized actions.
  • Privilege Escalation: Exploiting vulnerabilities to gain higher-level access or permissions than initially authorized.
  • Forgery: Creating fake data or credentials to deceive a system or user.
  • Directory Traversal: Exploiting vulnerabilities to access files and directories outside of the intended web root directory.

Cryptographic Attacks

  • Downgrade: Forcing a system to use a weaker, less secure version of a protocol or algorithm.
  • Collision: Finding two different inputs that produce the same hash output, undermining the integrity of hashing algorithms.
  • Birthday: A type of cryptographic attack that exploits the mathematics behind the birthday problem in probability theory to find collisions in hash functions.

Password Attacks

  • Spraying: Attempting a single common password against many accounts to avoid account lockouts.
  • Brute Force: Systematically trying every possible password combination until the correct one is found.

Indicators

  • Account Lockout: Frequent or sudden lockouts of user accounts, indicating potential brute-force attempts.
  • Concurrent Session Usage: A single user account being logged in from multiple locations simultaneously, suggesting compromise.
  • Blocked Content: Legitimate websites or content being blocked by security systems, potentially due to malware or misconfiguration.
  • Impossible Travel: A user logging in from two geographically distant locations within an impossibly short time frame.
  • Resource Consumption: Unusual spikes in CPU, memory, disk I/O, or network bandwidth usage, indicating malware activity or DDoS attacks.
  • Resource Inaccessibility: Legitimate users unable to access resources, potentially due to a denial-of-service attack or system compromise.
  • Out-of-cycle Logging: Logs appearing at unusual times or containing unexpected entries.
  • Published/Documented: Indicators of compromise (IoCs) that have been publicly shared or documented by security researchers.
  • Missing Logs: Gaps in log files, suggesting an attacker has tampered with or deleted logs to hide their tracks.
