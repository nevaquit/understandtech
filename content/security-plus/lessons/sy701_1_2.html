Summarize fundamental security concepts

Understanding fundamental security concepts is essential for building a strong foundation in cybersecurity. These concepts form the basis for all security decisions and implementations in an organization.

CIA Triad

The CIA Triad represents the three core principles of information security. Every security control and decision should be evaluated against these three pillars.

Confidentiality Confidentiality ensures that sensitive information is accessed only by authorized individuals, entities, or processes. This principle protects information from unauthorized disclosure and maintains privacy.

Key aspects of confidentiality include:

  • Data classification schemes that categorize information based on sensitivity levels
  • Access controls that restrict who can view specific information
  • Encryption that protects data both at rest and in transit
  • Need-to-know principles that limit access to only what is necessary for job functions
  • Non-disclosure agreements (NDAs) that legally bind individuals to protect sensitive information

Common threats to confidentiality:

  • Unauthorized access to systems or data
  • Social engineering attacks that trick users into revealing information
  • Data breaches that expose sensitive information
  • Insider threats from malicious or careless employees
  • Weak encryption or poor key management

Integrity Integrity ensures the accuracy, completeness, and reliability of information and systems. It protects against unauthorized modification, deletion, or corruption of data and ensures that information remains trustworthy.

Key aspects of integrity include:

  • Data validation mechanisms that verify information accuracy
  • Digital signatures that prove data hasn't been tampered with
  • Hash functions that detect unauthorized changes to files
  • Version control systems that track changes to documents and code
  • Checksums that verify data transmission accuracy
  • Access controls that prevent unauthorized modifications

Common threats to integrity:

  • Malware that corrupts or modifies data
  • Unauthorized changes by malicious insiders
  • System errors that corrupt data
  • Man-in-the-middle attacks that alter data in transit
  • Poor change management processes

Availability Availability ensures that data, applications, and services are accessible to authorized users when needed. This principle focuses on maintaining operational systems and preventing service disruptions.

Key aspects of availability include:

  • Redundant systems that provide backup capabilities
  • Load balancing that distributes traffic across multiple servers
  • Disaster recovery plans that restore operations after incidents
  • Business continuity planning that maintains operations during disruptions
  • Regular maintenance and updates to prevent system failures
  • Monitoring systems that detect and alert on service issues

Common threats to availability:

  • Denial-of-Service (DoS) and Distributed Denial-of-Service (DDoS) attacks
  • Hardware failures that cause system outages
  • Natural disasters that damage infrastructure
  • Power outages that disrupt operations
  • Network connectivity issues
  • Ransomware attacks that encrypt critical systems

Additional Security Concepts

Non-repudiation Non-repudiation provides proof of the origin, authenticity, and integrity of data, preventing a sender from denying they sent a message or performed an action. This concept is crucial for legal and business accountability.

Implementation methods:

  • Digital signatures that cryptographically bind actions to specific individuals
  • Audit logs that record user activities with timestamps
  • Biometric authentication that uniquely identifies individuals
  • Trusted timestamping services that prove when actions occurred
  • Certificate authorities that validate digital identities

Authentication, Authorization, and Accounting (AAA) AAA is a framework for controlling access to computer resources, enforcing policies, auditing usage, and providing information necessary to bill for services.

Authentication Authentication verifies the identity of a user, device, or system attempting to access resources. It answers the question "Who are you?"

Types of authentication factors:

  • Something you know (passwords, PINs, security questions)
  • Something you have (smart cards, tokens, mobile devices)
  • Something you are (biometrics like fingerprints, retina scans)
  • Something you do (behavioral patterns, typing rhythms)
  • Somewhere you are (location-based authentication)

Authorization Authorization determines what an authenticated user is allowed to do. It answers the question "What are you allowed to access?"

Authorization models include:

  • Role-Based Access Control (RBAC) that assigns permissions based on job roles
  • Attribute-Based Access Control (ABAC) that uses multiple attributes for decisions
  • Mandatory Access Control (MAC) that uses security labels and clearances
  • Discretionary Access Control (DAC) that allows owners to control access

Accounting (Auditing) Accounting tracks user activities and resource consumption for security monitoring, billing, and compliance purposes. It answers the question "What did you do?"

Accounting includes:

  • Logging user login/logout times and activities
  • Tracking resource usage and access attempts
  • Monitoring system changes and administrative actions
  • Generating reports for compliance and security analysis

Gap Analysis Gap analysis is a systematic process of comparing current security posture against desired security standards or requirements. It identifies areas where security controls are missing, inadequate, or improperly implemented.

Steps in gap analysis: 1. Define security requirements and standards 2. Assess current security controls and practices 3. Identify gaps between current state and requirements 4. Prioritize gaps based on risk and impact 5. Develop remediation plans to address identified gaps 6. Implement corrective measures and monitor progress

Zero Trust Architecture Zero Trust is a security model that assumes no implicit trust for any entity, whether inside or outside the network perimeter. It requires strict verification for every person and device trying to access resources.

Core principles of Zero Trust:

  • "Never trust, always verify" - all access requests must be authenticated and authorized
  • Least privilege access - users receive minimum necessary permissions
  • Assume breach - design systems assuming attackers are already present
  • Verify explicitly - use multiple data sources for access decisions
  • Continuous monitoring - constantly assess security posture and user behavior

Zero Trust implementation includes:

  • Multi-factor authentication for all access requests
  • Micro-segmentation to limit lateral movement
  • Encryption of all communications
  • Continuous monitoring and analytics
  • Identity and access management (IAM) systems
  • Software-defined perimeters (SDP)

Physical Security Physical security protects personnel, hardware, software, networks, and data from physical actions and events that could cause serious loss or damage. It forms the foundation of overall security strategy.

Physical security components include:

*Perimeter Security:*

  • Fencing and barriers to define property boundaries
  • Gates and access points with controlled entry
  • Lighting systems to deter intruders and aid surveillance
  • Landscaping designed to eliminate hiding places

*Building Security:*

  • Access control systems (card readers, biometric scanners)
  • Security guards and reception areas
  • Visitor management systems
  • Mantrap/airlock systems for high-security areas

*Environmental Controls:*

  • Fire detection and suppression systems
  • HVAC systems to maintain proper temperature and humidity
  • Power systems including UPS and backup generators
  • Water detection systems to prevent flooding damage

*Surveillance Systems:*

  • CCTV cameras with recording capabilities
  • Motion detection systems
  • Alarm systems for unauthorized access
  • Security monitoring centers

Deception and Disruption Technology These technologies are designed to mislead attackers, gather intelligence about attack methods, and disrupt malicious activities.

Honeypots Honeypots are decoy systems designed to attract and trap attackers. They appear to be legitimate systems but are actually isolated and monitored to study attack techniques.

Types of honeypots:

  • Low-interaction honeypots that simulate services and capture basic attack data
  • High-interaction honeypots that provide full operating systems for detailed analysis
  • Production honeypots deployed in live networks to detect real attacks
  • Research honeypots used to study new attack techniques and malware

Honeynets Honeynets are networks of honeypots that create more realistic environments for attackers. They provide comprehensive views of attack campaigns and allow researchers to study coordinated attacks.

Honeyfiles Honeyfiles are decoy documents placed in file systems to detect unauthorized access. When accessed, they trigger alerts indicating potential compromise.

Honeytokens Honeytokens are pieces of data (credentials, URLs, database records) that have no legitimate use but trigger alerts when accessed, indicating unauthorized activity.

🔒 Authorization Models - Detailed Comparison

👔 Role-Based Access Control (RBAC)

Permissions through role assignment:

  • Users assigned to roles
  • Roles granted specific permissions
  • Hierarchical role inheritance
  • Simplifies large organization management

🏛️ Mandatory Access Control (MAC)

Central authority enforcement:

  • Security labels classify data
  • Clearance levels for users
  • Non-discretionary access decisions
  • Common in government/military

👤 Discretionary Access Control (DAC)

Owner-controlled permissions:

  • Resource owners set permissions
  • Flexible but potentially inconsistent
  • Identity-based access control
  • Common in commercial environments

📋 Rule-Based Access Control

Conditional access enforcement:

  • Access based on predefined rules
  • Time, location, and context factors
  • System state considerations
  • Dynamic policy enforcement

🏷️ Attribute-Based Access Control (ABAC)

Multi-factor decision making:

  • Subject, object, action attributes
  • Environmental context factors
  • Dynamic policy evaluation
  • Most flexible and granular model

Gap analysis A common task performed on a recurring basis, and often in preparation for external audits is the gap analysis. Auditors will follow a standard (often ISO 27001) and then compare standard requirements to the org’s current operations. Deficiencies versus the standard will be captured in the audit report as gaps, sometimes called control gaps. Control gap a discrepancy between the security measures an organization should have in place versus controls actually in place. All auditors should have independence, but attestations from external auditors carry more weight (higher confidence) The outcome of an audit is an attestation, which is a formal statement made by the auditor on controls and processes in place.

Access policy enforcement Policy Enforcement Point responsible for enabling, monitoring, and terminating connections between a subject (such as a user or device) and an enterprise resource. acts as the gateway that enforces access control policies. when an access request occurs, the PEP evaluates the request against predefined policies and applies the necessary controls. For example, PEP might enforce Multi-Factor Authentication (MFA) for access requests from unexpected locations. Policy Decision Point is where access decisions are made based on various factors such as user identity, device health, and risk assessment. evaluates the context of an access request and decides whether it should be allowed, denied, or subjected to additional controls. considers the 5 W’s (who, what, when, where, and why) In short, the PEP enforces policies at the connection level, while the PDP makes access decisions based on contextual information. Dynamic based on conditions/context
