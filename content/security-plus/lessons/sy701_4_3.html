Exam objective 4.3

Explain various activities associated with vulnerability management

Vulnerability management is a continuous process of identifying, assessing, prioritizing, and remediating vulnerabilities in systems and applications.

Identification Methods

  • Vulnerability Scan: Automated tools that scan systems and networks for known vulnerabilities.
  • Application Security:

- Static analysis (SAST): Analyzing source code without executing it to find vulnerabilities. - Dynamic analysis (DAST): Testing a running application for vulnerabilities by simulating attacks. - Package monitoring: Monitoring third-party libraries and components for known vulnerabilities.

  • Threat Feed: Subscriptions to services that provide real-time information about new threats and vulnerabilities.

- Open-source intelligence (OSINT): Gathering information from publicly available sources. - Proprietary/third-party: Commercial threat intelligence feeds. - Information-sharing organization: Organizations that facilitate the sharing of threat intelligence among members. - Dark web: Monitoring the dark web for mentions of organizational assets or new attack techniques.

  • Penetration Testing: Simulating real-world attacks to identify vulnerabilities and assess the effectiveness of security controls.
  • Responsible Disclosure Program: A program where security researchers can report vulnerabilities to an organization in a responsible manner.

- Bug bounty program: A program that offers rewards to security researchers for finding and reporting vulnerabilities.

  • System/Process Audit: Reviewing system configurations, logs, and processes to identify security weaknesses or non-compliance.

Analysis

  • Confirmation: Verifying that a reported vulnerability is real and not a false positive.

- False positive: A security alert that indicates a threat when none exists. - False negative: A security event that is not detected by security tools.

  • Prioritize: Ranking vulnerabilities based on their severity, exploitability, and potential impact.
  • Common Vulnerability Scoring System (CVSS): A standardized method for assigning severity scores to vulnerabilities.
  • Common Vulnerability Enumeration (CVE): A list of publicly disclosed cybersecurity vulnerabilities.
  • Vulnerability Classification: Categorizing vulnerabilities based on their type, affected system, or other criteria.
  • Exposure Factor: The percentage of an asset that would be lost if a specific threat were to materialize.
  • Environmental Variables: Factors in the environment that can influence the risk of a vulnerability (e.g., network configuration, existing security controls).
  • Industry/Organizational Impact: The potential impact of a vulnerability on the organization and its industry.
  • Risk Tolerance: The level of risk an organization is willing to accept.

Vulnerability Response and Remediation

  • Patching: Applying software updates and fixes to address vulnerabilities.
  • Insurance: Cyber insurance can help mitigate the financial impact of a security breach.
  • Segmentation: Isolating vulnerable systems to prevent the spread of attacks.
  • Compensating Controls: Implementing alternative controls to mitigate risk when primary controls are not feasible or effective.
  • Exceptions and Exemptions: Formal processes for documenting and approving deviations from security policies for specific vulnerabilities.

Validation of Remediation

  • Rescanning: Performing another vulnerability scan after remediation to confirm that the vulnerability has been fixed.
  • Audit: Conducting an audit to verify that remediation steps were followed and are effective.
  • Verification: Manually testing or confirming that the vulnerability is no longer present.
  • Reporting: Documenting the vulnerability management process, including identified vulnerabilities, remediation efforts, and current risk posture.
