Exam objective 1.4

Explain the importance of using appropriate cryptographic solutions

Cryptography is the practice and study of techniques for secure communication in the presence of third parties. It forms the foundation of modern information security by providing confidentiality, integrity, authentication, and non-repudiation. Understanding cryptographic solutions is essential for implementing effective security controls.

Public Key Infrastructure (PKI)

Public Key Infrastructure is a framework that manages digital keys and certificates for secure communication. PKI enables secure data exchange over insecure networks by providing authentication, encryption, and digital signature capabilities.

The PKI process ensures:

  • Identity verification before certificate issuance
  • Secure key generation and distribution
  • Certificate lifecycle management
  • Trust establishment through the CA hierarchy
  • Revocation capabilities for compromised certificates

Public Key A public key is part of an asymmetric key pair that can be freely shared without compromising security. It is mathematically related to its corresponding private key but cannot be used to derive the private key.

Public key uses:

  • Encrypting data that only the private key holder can decrypt
  • Verifying digital signatures created with the corresponding private key
  • Establishing secure communication channels
  • Authenticating the identity of key holders
  • Supporting key exchange protocols

Private Key A private key is the secret component of an asymmetric key pair that must be kept confidential by its owner. It is used for decryption and digital signature creation.

Private key characteristics:

  • Must be kept secret and secure at all times
  • Used to decrypt data encrypted with the corresponding public key
  • Used to create digital signatures that can be verified with the public key
  • Should be protected with strong access controls and encryption
  • May be stored in hardware security modules (HSMs) for additional protection

Key Escrow Key escrow is a system where cryptographic keys are held by a trusted third party (escrow agent) to ensure data recovery in specific circumstances, such as legal requirements or employee departure.

Key escrow considerations:

  • Legal and regulatory compliance requirements
  • Data recovery capabilities for business continuity
  • Trust and security of the escrow agent
  • Access controls and audit trails for key retrieval
  • Balance between security and accessibility

Encryption

Encryption is the process of converting plaintext into ciphertext using cryptographic algorithms and keys. It provides confidentiality by making data unreadable to unauthorized parties.

Levels and Types

*Full Disk Encryption (FDE):* Encrypts entire storage devices to protect data at rest. Examples include BitLocker (Windows) and FileVault (macOS).

*File-level Encryption:* Encrypts individual files or folders, allowing selective protection of sensitive data.

*Database Encryption:* Protects database contents through transparent data encryption (TDE) or column-level encryption.

*Application-level Encryption:* Implemented within applications to protect specific data elements or communications.

*Transport Encryption:* Protects data in transit using protocols like TLS/SSL, IPSec, or SSH.

Algorithms

*Symmetric Algorithms:* Use the same key for encryption and decryption. Examples include:

  • AES (Advanced Encryption Standard) - most widely used, supports 128, 192, and 256-bit keys
  • DES (Data Encryption Standard) - legacy algorithm, no longer considered secure
  • 3DES (Triple DES) - applies DES three times, being phased out
  • ChaCha20 - modern stream cipher used in TLS and VPNs

*Asymmetric Algorithms:* Use different keys for encryption and decryption. Examples include:

  • RSA - widely used for key exchange and digital signatures
  • ECC (Elliptic Curve Cryptography) - provides equivalent security with smaller key sizes
  • Diffie-Hellman - used for key exchange protocols
  • DSA (Digital Signature Algorithm) - specifically designed for digital signatures

The following diagram illustrates the fundamental differences between symmetric and asymmetric cryptographic algorithms:

This comparison highlights the key operational differences:

Symmetric Encryption:

  • Uses a single shared key for both encryption and decryption
  • Both parties must securely share the same key beforehand
  • Faster and more efficient for large amounts of data
  • Primary use case: protecting data at rest (stored data)
  • Examples: AES, DES, 3DES, ChaCha20

Asymmetric Encryption:

  • Uses a mathematically related key pair (public and private keys)
  • Public key can be freely shared, private key must be kept secret
  • Slower but enables secure communication without prior key sharing
  • Primary use case: protecting data in transit and key exchange
  • Examples: RSA, ECC, Diffie-Hellman

Practical Implementation: Most modern systems use hybrid cryptography, combining both approaches: 1. Asymmetric encryption securely exchanges symmetric keys 2. Symmetric encryption handles the actual data encryption 3. This provides both security and performance benefits

Key Exchange Key exchange protocols allow parties to securely establish shared cryptographic keys over insecure channels.

Common key exchange methods:

  • Diffie-Hellman Key Exchange - allows two parties to establish a shared secret
  • RSA Key Exchange - uses RSA encryption to securely transmit symmetric keys
  • Elliptic Curve Diffie-Hellman (ECDH) - more efficient version using elliptic curves
  • Perfect Forward Secrecy (PFS) - ensures session keys cannot be compromised even if long-term keys are compromised

Key Length Key length determines the strength of cryptographic algorithms. Longer keys generally provide better security but require more computational resources.

Recommended key lengths:

  • AES: 256-bit keys for top secret data, 128-bit for general use
  • RSA: 2048-bit minimum, 3072-bit or 4096-bit for high security
  • ECC: 256-bit provides equivalent security to 3072-bit RSA
  • Hash functions: SHA-256 or SHA-3 for general use

Cryptographic Tools and Concepts

Trusted Platform Module (TPM) TPM is a hardware-based security chip that provides secure storage for cryptographic keys, passwords, and certificates. It offers hardware-based root of trust for system integrity.

TPM capabilities:

  • Secure key generation and storage
  • Platform authentication and attestation
  • Measured boot process verification
  • Hardware-based random number generation
  • Cryptographic operations in secure hardware

Hardware Security Module (HSM) HSM is a dedicated cryptographic device that provides secure key management and cryptographic operations. HSMs offer higher security than software-based solutions.

HSM benefits:

  • Tamper-resistant hardware protection
  • High-performance cryptographic operations
  • Secure key generation and storage
  • Role-based authentication and access controls
  • Compliance with security standards (FIPS 140-2, Common Criteria)

Key Management System A key management system provides centralized control over cryptographic keys throughout their lifecycle, from generation to destruction.

Key management functions:

  • Key generation using secure random number generators
  • Key distribution to authorized parties
  • Key storage with appropriate protection
  • Key rotation and renewal procedures
  • Key revocation and destruction
  • Audit logging and compliance reporting

Secure Enclave A secure enclave is a protected area within a processor that provides hardware-based security for sensitive operations and data.

Secure enclave features:

  • Isolated execution environment
  • Hardware-based memory protection
  • Secure storage for sensitive data
  • Cryptographic operations in protected space
  • Attestation capabilities for remote verification

Obfuscation Obfuscation makes code or data difficult to understand or reverse-engineer while maintaining functionality. It provides security through obscurity but should not be the primary security mechanism.

Obfuscation techniques:

  • Code obfuscation to hide software logic
  • Data obfuscation to protect sensitive information
  • Network traffic obfuscation to hide communication patterns
  • Database obfuscation for development and testing environments

Hashing Hashing creates a fixed-size digest from variable-length input data. Hash functions are one-way operations used for data integrity verification and password storage.

Hash function properties:

  • Deterministic - same input always produces same output
  • Fixed output size regardless of input size
  • Avalanche effect - small input changes cause large output changes
  • Irreversible - computationally infeasible to reverse
  • Collision resistant - difficult to find two inputs with same output

Common hash algorithms:

  • SHA-256 and SHA-3 for general cryptographic use
  • MD5 - legacy algorithm, no longer considered secure
  • BLAKE2 - high-performance alternative to SHA-2
  • bcrypt and scrypt for password hashing

Salting Salting adds random data to passwords before hashing to prevent rainbow table attacks and ensure unique hashes for identical passwords.

Salt characteristics:

  • Should be unique for each password
  • Should be randomly generated
  • Should be stored alongside the hash
  • Increases computational cost for attackers
  • Prevents precomputed hash attacks

Digital Signatures Digital signatures provide authentication, integrity, and non-repudiation for digital documents and messages using asymmetric cryptography.

Digital signature process: 1. Hash the document to create a digest 2. Encrypt the hash with the sender's private key 3. Attach the encrypted hash (signature) to the document 4. Recipients verify by decrypting with sender's public key 5. Compare decrypted hash with newly computed document hash

Key Stretching Key stretching increases the computational cost of password-based attacks by applying cryptographic functions multiple times.

Key stretching methods:

  • PBKDF2 (Password-Based Key Derivation Function 2)
  • bcrypt - incorporates salt and adaptive cost parameter
  • scrypt - memory-hard function that resists hardware attacks
  • Argon2 - winner of password hashing competition, resistant to various attacks

Blockchain Blockchain is a distributed ledger technology that uses cryptographic hashing to create an immutable record of transactions.

Blockchain security features:

  • Cryptographic hashing links blocks together
  • Distributed consensus prevents unauthorized changes
  • Digital signatures authenticate transactions
  • Merkle trees efficiently verify data integrity
  • Proof-of-work or proof-of-stake consensus mechanisms

Certificates Digital certificates bind public keys to identities and are issued by trusted Certificate Authorities (CAs). They enable secure communication and authentication in PKI systems.

Certificate components:

  • Subject information (name, organization, location)
  • Public key and algorithm information
  • Issuer (Certificate Authority) information
  • Validity period (not before/not after dates)
  • Digital signature from the issuing CA
  • Certificate extensions for additional functionality

Certificate types:

  • Domain Validation (DV) - verifies domain ownership
  • Organization Validation (OV) - verifies organization identity
  • Extended Validation (EV) - highest level of validation
  • Wildcard certificates - cover multiple subdomains
  • Code signing certificates - verify software authenticity
