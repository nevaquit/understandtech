Exam objective 4.4

Explain security alerting and monitoring concepts and tools

Security alerting and monitoring are crucial for detecting and responding to security incidents in a timely manner. This involves continuous observation of computing resources and the use of specialized tools.

Monitoring Computing Resources

  • Systems: Monitoring operating systems for suspicious processes, file changes, and system events.
  • Applications: Monitoring application logs, performance, and user activity for anomalies.
  • Infrastructure: Monitoring network devices, servers, and cloud infrastructure for security events and performance issues.

Activities

  • Log Aggregation: Collecting logs from various sources (e.g., firewalls, servers, applications) into a central repository for analysis.
  • Alerting: Generating notifications when predefined security events or thresholds are met.
  • Scanning: Regularly scanning systems and networks for vulnerabilities and misconfigurations.
  • Reporting: Generating reports on security posture, incidents, and compliance.
  • Archiving: Storing logs and security data for long-term retention, forensic analysis, and compliance.
  • Alert response and remediation/validation: The process of responding to security alerts, taking corrective actions, and verifying that the issue has been resolved.

Tools

  • Security Content Automation Protocol (SCAP): A suite of specifications that standardize how security software communicates information about security flaws and configurations.
  • Benchmarks: Security configuration guidelines and best assess and harden systems.
  • Agents/Agentless:
    • Agents: Software installed on endpoints to collect security data and enforce policies.
    • Agentless: Monitoring solutions that collect data remotely without installing software on the target system.
  • Security Information and Event Management (SIEM): A system that aggregates and analyzes log data from various sources to provide real-time security monitoring, threat detection, and incident response capabilities.
  • Antivirus: Software designed to detect, prevent, and remove malicious software.
    • Quarantine: Isolating suspicious files to prevent them from causing harm.
    • Alert tuning: Adjusting alert rules to reduce false positives and focus on critical threats.
  • Data Loss Prevention (DLP): Systems that identify, monitor, and protect sensitive data to prevent unauthorized disclosure.
  • Simple Network Management Protocol (SNMP) traps: Notifications sent by network devices to a central management system when specific events occur.
  • NetFlow: A network protocol that collects IP traffic information as it enters or exits an interface, used for network monitoring and analysis.
  • Vulnerability scanners: Tools used to identify security weaknesses in systems and applications.

4.4 NETFLOW A feature available on many routers and switches that can collect IP traffic statistics and send them to a NetFlow collector. The NetFlow collector receives and stores the data. Protocol analyzers like Wireshark enable capture and view of data, including headers and payloads of individual packets. By comparison, NetFlow only records counts or stats related to data a device receives. Uses templates to identify what data to include in a NetFlow packet, but generally includes: Output interface identifier (will be zero if a packet is dropped) Source information (source IP address and port number, if used) Timestamps identifying the start and finish time of the flow Input interface identifier (on router or switch)
