Exam objective 2.5

Explain the purpose of mitigation techniques used to secure the enterprise

Mitigation techniques are strategies and controls implemented to reduce the likelihood or impact of a security threat or vulnerability. They are essential for building a robust security posture.

Mitigation Techniques

  • Segmentation: Dividing a network into smaller, isolated segments to limit the spread of attacks. This can be done through VLANs, firewalls, or other network devices.
  • Access Control: Restricting access to resources based on user identity and permissions. This includes physical access controls (e.g., badges, locks) and logical access controls (e.g., user accounts, permissions).

- Access Control List (ACL): A list of permissions associated with a system resource (e.g., file, directory, network port) that specifies which users or system processes are granted access to objects, as well as what operations are allowed on given objects. - Permissions: The specific rights granted to a user or group to perform actions on a resource (e.g., read, write, execute, delete).

  • Application Allow List (Whitelisting): A security measure that allows only approved applications to run on a system, blocking all others. This is a highly effective preventive control against malware.
  • Isolation: Separating critical systems or sensitive data from less secure environments to minimize the attack surface and prevent lateral movement by attackers. This can involve network isolation, virtual machine isolation, or sandboxing.
  • Patching: Applying updates and fixes to software, operating systems, and firmware to address known vulnerabilities. Regular and timely patching is critical for maintaining security.
  • Encryption: Transforming data into a coded format to prevent unauthorized access. Used for data at rest (e.g., full disk encryption), data in transit (e.g., TLS, VPNs), and data in use (e.g., secure enclaves).
  • Monitoring: Continuously observing systems, networks, and user activities for signs of malicious behavior or security incidents. This involves collecting logs, analyzing traffic, and using security tools.
  • Least Privilege: Granting users or systems only the minimum necessary permissions to perform their tasks. This reduces the potential damage if an account is compromised.
  • Configuration Enforcement: Ensuring that systems and applications adhere to predefined secure configurations and policies. This can involve automated tools for configuration management and compliance checking.
  • Decommissioning: Securely removing systems, applications, or data that are no longer needed. This includes sanitization (wiping data) and destruction of hardware to prevent data leakage.
  • Hardening Techniques: Implementing various measures to reduce the attack surface and strengthen the security of systems and applications.

- Encryption: Encrypting data to protect its confidentiality. - Installation of endpoint protection: Deploying antivirus, anti-malware, and endpoint detection and response (EDR) solutions. - Host-based firewall: A firewall running on an individual host to control network traffic to and from that specific device. - Host-based intrusion prevention system (HIPS): Software that monitors a host for suspicious activity and can block or alert on detected threats. - Disabling ports/protocols: Turning off unnecessary network ports and protocols to reduce the attack surface. - Default password changes: Changing default credentials immediately after installation to prevent unauthorized access. - Removal of unnecessary software: Uninstalling applications, services, or features that are not required for the system's function to reduce potential vulnerabilities.
