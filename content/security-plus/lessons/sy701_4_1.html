Exam objective 4.1

Given a scenario, apply common security techniques to computing resources

Applying security techniques to computing resources involves establishing secure baselines, hardening targets, and implementing various security measures for different types of devices and applications.

  • Establish: Defining a secure configuration for systems and applications based on security policies, industry best practices, and compliance requirements.
  • Deploy: Implementing the established baselines across the organization's computing resources, often using automated tools.
  • Maintain: Regularly reviewing and updating baselines to adapt to new threats, vulnerabilities, and changes in the IT environment.

Hardening Targets

  • Mobile Devices: Securing smartphones, tablets, and other mobile devices through MDM, strong authentication, encryption, and secure app usage.
  • Workstations: Implementing security measures on end-user computers, including antivirus, firewalls, patch management, and user access controls.
  • Switches: Securing network switches by disabling unused ports, implementing port security, and using secure management protocols.
  • Routers: Securing network routers by changing default credentials, disabling unnecessary services, and configuring access control lists (ACLs).
  • Cloud Infrastructure: Applying security best practices to cloud-based resources, including identity and access management, network security groups, and data encryption.
  • Servers: Hardening servers by removing unnecessary services, applying patches, configuring firewalls, and implementing strong authentication.
  • ICS/SCADA: Securing industrial control systems and SCADA environments, which often require specialized security measures due to their operational nature and potential for physical impact.
  • Embedded Systems: Securing specialized computer systems embedded within other devices, often with limited resources and unique security challenges.
  • RTOS (Real-time Operating System): Securing operating systems designed for applications with strict timing requirements, where security measures must not compromise real-time performance.
  • IoT Devices: Securing Internet of Things devices, which often have limited security features, default credentials, and are vulnerable to various attacks.

Wireless Devices

  • Installation Considerations:
    • Site surveys: Conducting surveys to determine optimal placement of wireless access points for coverage and security.
    • Heat maps: Visual representations of wireless signal strength, used to identify areas of weak coverage or potential signal leakage.
  • Mobile Solutions:
    • Mobile Device Management (MDM): Software that allows organizations to manage, monitor, and secure mobile devices used by employees.
    • Deployment models:
      • Bring Your Own Device (BYOD): Employees use their personal devices for work, requiring strong security policies and MDM.
      • Corporate-owned, Personally Enabled (COPE): Organization owns the device, but employees can use it for personal activities, with security policies in place.
      • Choose Your Own Device (CYOD): Employees choose from a list of approved devices provided by the organization.
    • Connection methods: Securing various mobile connection methods.
      • Cellular: Securing data transmission over cellular networks.
      • Wi-Fi: Implementing strong Wi-Fi security protocols (e.g., WPA3).
      • Bluetooth: Securing Bluetooth connections to prevent unauthorized access.
  • Wireless Security Settings:
    • Wi-Fi Protected Access 3 (WPA3): The latest security standard for Wi-Fi networks, offering stronger encryption and improved authentication.
    • AAA/Remote Authentication Dial-In User Service (RADIUS): Centralized authentication, authorization, and accounting for network access.
    • Cryptographic protocols: Using strong encryption protocols for wireless communication.
    • Authentication protocols: Implementing secure authentication methods for wireless access.
  • Application Security:
    • Input validation: Ensuring that all user input is properly sanitized and validated to prevent injection attacks.
    • Secure cookies: Using secure flags (e.g., HttpOnly, Secure) for cookies to prevent cross-site scripting (XSS) and ensure transmission over HTTPS.
    • Static code analysis: Analyzing source code without executing it to find vulnerabilities.
    • Code signing: Digitally signing software to verify its authenticity and integrity.
  • Sandboxing: Running applications in an isolated environment to prevent them from accessing or harming the rest of the system.
  • Monitoring: Continuously observing systems and networks for suspicious activity and security events.
