Exam objective 3.2

Given a scenario, apply security principles to secure enterprise infrastructure

Securing enterprise infrastructure involves applying various security principles and technologies to protect systems and data.

Infrastructure Considerations

  • Device Placement: Strategic positioning of network devices (e.g., firewalls, routers) to create security zones and control traffic flow.
  • Security Zones: Logical or physical segments of a network with different security requirements and controls (e.g., DMZ, internal network).
  • Attack Surface: Reducing the number of entry points or vulnerabilities that an attacker can exploit.
  • Connectivity: Securely managing network connections, both internal and external.
  • Failure Modes: Understanding how systems behave when they fail and designing for secure failure states.
    • Fail-open: System fails in a state that allows access (e.g., a firewall failing open and allowing all traffic). Generally undesirable for security.
    • Fail-closed: System fails in a state that denies access (e.g., a firewall failing closed and blocking all traffic). Generally preferred for security.
  • Device Attributes:
    • Active vs. Passive:
      • Active: Devices that actively intervene to prevent or mitigate threats (e.g., IPS).
      • Passive: Devices that monitor and detect threats without actively intervening (e.g., IDS).
    • Inline vs. Tap/Monitor:
      • Inline: Devices placed directly in the network path, allowing them to block or modify traffic.
      • Tap/Monitor: Devices that receive a copy of network traffic for analysis without being in the direct path.
  • Network Appliances:
    • Jump server: A hardened server used as an intermediary to access sensitive network segments, providing a controlled and monitored access point.
    • Proxy server: A server that acts as an intermediary for requests from clients seeking resources from other servers. Can be used for content filtering, caching, and anonymity.
    • Intrusion Prevention System (IPS)/Intrusion Detection System (IDS):
      • IDS: Detects suspicious activity and alerts administrators.
      • IPS: Detects and actively blocks suspicious activity.
    • Load balancer: Distributes network traffic across multiple servers to improve performance, availability, and resilience. Can also provide some security features.
    • Sensors: Devices or software that collect data for security monitoring (e.g., network sensors, host-based agents).
    • Port security: Limiting the number of MAC addresses that can connect to a switch port to prevent unauthorized devices.
    • 802.1X: A port-based network access control standard that provides an authentication mechanism for devices trying to connect to a network.
    • Extensible Authentication Protocol (EAP): A framework for authentication often used with 802.1X.
    • Firewall types:
      • Web Application Firewall (WAF): Protects web applications from common web-based attacks (e.g., SQL injection, XSS).
      • Unified Threat Management (UTM): An all-in-one security appliance that combines multiple security functions (e.g., firewall, IPS, antivirus, content filtering).
      • Next-generation firewall (NGFW): Firewalls that go beyond traditional port/protocol inspection to include application awareness, intrusion prevention, and threat intelligence.
      • Layer 4/Layer 7: Refers to the OSI model layers at which a firewall operates. Layer 4 (transport layer) firewalls inspect TCP/UDP ports; Layer 7 (application layer) firewalls inspect application-level traffic.

Secure Communication/Access

  • Virtual Private Network (VPN): Creates a secure, encrypted connection over a public network, allowing remote users to securely access internal network resources.
  • Remote Access: Securely enabling users to connect to internal systems from external locations.
  • Tunneling: Encapsulating one network protocol within another, often used in VPNs to create secure communication channels.
  • Transport Layer Security (TLS): Cryptographic protocol designed to provide communications security over a computer network. Widely used for secure web browsing (HTTPS).
  • Internet Protocol Security (IPSec): A suite of protocols for securing Internet Protocol (IP) communications by authenticating and encrypting each IP packet of a communication session.
  • Software-defined Wide Area Network (SD-WAN): An approach to managing a wide area network that uses software-defined networking principles to route traffic more efficiently and securely.
  • Secure Access Service Edge (SASE): A cloud-native security model that combines network security functions (e.g., firewall, CASB, secure web gateway) with WAN capabilities into a single, cloud-delivered service.

Selection of Effective Controls

  • Choosing the right security controls based on risk assessment, compliance requirements, and the specific threats and vulnerabilities faced by the organization.

🎯 Fundamental Secure Design Principles

  • Least Privilege: Grant minimum necessary access rights to users and systems
  • Defense in Depth: Implement multiple overlapping layers of security controls
  • Fail Secure/Safe: Systems default to secure state when encountering failures
  • Separation of Duties: Divide critical tasks among multiple individuals
  • Keep It Simple (KISS): Reduce complexity to minimize attack surface and errors
  • Zero Trust Architecture: Continuous verification with no implicit trust
  • Privacy by Design: Build privacy protection into systems from inception
  • Secure by Default: Ship systems with secure default configurations
