Exam focus

Given a scenario, use network monitoring technologies

  • MIB (Management Information Base): A database of objects that can be managed via SNMP.
  • SNMP Trap: An unsolicited alert sent from a managed device to the SNMP manager when a specific event occurs.
  • Community String: A password-like string used for SNMPv1/v2c authentication (default: "public" for read, "private" for write).
  • Always use SNMPv3 in production. SNMPv1 and v2c send community strings in cleartext, making them vulnerable to eavesdropping. Log Management and Other Monitoring Methods • Syslog: Standard protocol for sending log messages from network devices to a central server (port 514 UDP). • SIEM (Security Information and Event Management): Aggregates, correlates, and analyzes log data from multiple sources to identify security threats. • NetFlow/sFlow: Collects IP traffic information and monitors network flow data for traffic analysis. • Port Mirroring (SPAN): Copies traffic from one or more ports to a monitoring port for analysis. • Baseline Metrics: Establishing normal performance levels to identify anomalies. • API Integration: Using REST APIs to pull metrics directly from modern network devices and cloud services. • Availability Monitoring: Checking if a device is up (e.g., via ping/ICMP). • Configuration Monitoring: Alerting when a device configuration changes from the baseline.
