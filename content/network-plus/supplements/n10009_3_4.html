Exam focus

Given a scenario, implement IPv4 and IPv6 network services

  • DORA Process: Discover, Offer, Request, Acknowledge.
  • Scope: The range of IP addresses a DHCP server can hand out.
  • Exclusions: Specific IP addresses within the scope that the server is told NOT to hand out (reserved for static
  • Options: Additional info given to clients (Default Gateway, DNS servers, domain name).
  • DHCP Reservation: Assigns a specific IP address to a device based on its MAC address.
  • DHCP Relay/IP Helper: Forwards DHCP broadcasts across routers to a DHCP server on a different subnet.
  • SLAAC: IPv6 mechanism allowing devices to configure their own IP addresses without a DHCP server.
  • Forward Zone: Resolves names to IP addresses.
  • Reverse Zone: Resolves IP addresses to names (PTR records).
  • Authoritative Server: Holds the actual DNS records for a domain. The definitive source.
  • Non-Authoritative Server: Caches records from other servers; answers may be stale.
  • Primary vs. Secondary: Primary holds the master read/write copy; Secondary holds a read-only backup.
  • Recursive Server: Queries other servers on behalf of the client until it finds the answer.
  • Hosts File: A local text file that manually maps IPs to names. Checked before DNS queries.
  • DNS Security: DNSSEC adds digital signatures to DNS records to prevent DNS poisoning. DNS over HTTPS (DoH) and DNS over TLS (DoT) encrypt DNS queries to prevent eavesdropping.
