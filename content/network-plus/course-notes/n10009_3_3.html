Critical DR Metrics

Metric Definition Key Question

RPO (Recovery Point

Objective)

Maximum acceptable data loss (measured in time) "How much data can we afford to lose?" Metric Definition Key Question

RTO (Recovery Time

Objective)

Maximum acceptable downtime "How fast must we be back online?" MTTR (Mean Time to Repair) Average time to repair a failed component "How long does it take to fix?"

MTBF (Mean Time Between

Failures)

Average time between failures "How reliable is the system?"

Exam tip

RPO is about data loss; RTO is about downtime. A lower RPO requires more frequent backups. A lower RTO requires more redundant infrastructure (like a hot site). DR Site Types Site Type Equipment Data Cost Recovery Time Cold Site None (empty facility) None (restore from backup) Low Days to weeks Warm Site Some equipment pre-staged Periodic backups Medium Hours to days Hot Site Fully operational Real-time or near-real-time sync High Minutes to hours DR Testing Methods • Tabletop Exercises: Discussion-based walkthrough of the DR plan with key personnel. No systems are actually failed over. • Validation Tests: Actually failing over systems to verify the DR plan works technically. High Availability (HA) Approaches • Active-Active: Both nodes handle traffic simultaneously. If one fails, the other absorbs the load. Higher performance. • Active-Passive: One node handles traffic; the other is on standby. Simpler but less efficient.
