Feature Static Routing Dynamic Routing

Configuration Manual by admin Automatic via protocol

Scalability Poor (large networks) Excellent

Overhead None Uses bandwidth and CPU

Use Case Small networks, stub routes Large, complex networks

Dynamic Routing Protocols

Protocol Type Algorithm Metric Use Case

RIP Distance-vector Bellman-Ford Hop count (max 15) Small, legacy networks OSPF Link-state Dijkstra (SPF) Cost (bandwidth) Large enterprise networks

EIGRP Advanced distance-vector

DUAL Bandwidth + Delay Cisco environments

BGP Path-vector Best Path AS Path, Policy Internet routing (ISPs) Administrative Distance (AD) — Route Selection Priority When a router has multiple routes to the same destination, it uses Administrative Distance to choose the best one.

Lower AD = more trusted source.

Route Source Administrative Distance

Directly Connected 0

Static Route 1

Exam tip

If two routes have the same AD, the router uses the metric to choose. If both AD and metric are equal, the router will load-balance across both paths. Address Translation • NAT (Network Address Translation): Translates private IP addresses to a public IP address. • PAT (Port Address Translation): Maps multiple private IPs to a single public IP using different port numbers. Also called "NAT Overload." This is what your home router does. First Hop Redundancy Protocol (FHRP) Provides a virtual default gateway shared between multiple routers, ensuring clients can still reach the internet if one router fails. • HSRP (Hot Standby Router Protocol): Cisco proprietary. • VRRP (Virtual Router Redundancy Protocol): Open standard.
