SNMP (Simple Network Management Protocol)

Version Security Notes

SNMPv1 None Community strings in cleartext

SNMPv2c None Community strings in cleartext; improved performance SNMPv3 Authentication + Encryption Use this in production environments

  • MIB (Management Information Base): A database of objects that can be managed via SNMP.
  • SNMP Trap: An unsolicited alert sent from a managed device to the SNMP manager when a specific event occurs.
  • Community String: A password-like string used for SNMPv1/v2c authentication (default: "public" for read, "private" for write).

Log Management and Other Monitoring Methods

  • Syslog: Standard protocol for sending log messages from network devices to a central server (port 514 UDP).
  • SIEM (Security Information and Event Management): Aggregates, correlates, and analyzes log data from multiple sources to identify security threats.
  • NetFlow/sFlow: Collects IP traffic information and monitors network flow data for traffic analysis.
  • Port Mirroring (SPAN): Copies traffic from one or more ports to a monitoring port for analysis.
  • Baseline Metrics: Establishing normal performance levels to identify anomalies.
  • API Integration: Using REST APIs to pull metrics directly from modern network devices and cloud services.
  • Availability Monitoring: Checking if a device is up (e.g., via ping/ICMP).
  • Configuration Monitoring: Alerting when a device configuration changes from the baseline.

Exam focus

Given a scenario, use network monitoring technologies

  • SNMPv3 adds auth and privacy; avoid SNMPv1 community strings in production.
  • Syslog centralizes events; SIEM adds correlation and alerting.
  • Port mirroring copies traffic to analysis tools.
  • Baselines help spot anomalies versus normal utilization.
