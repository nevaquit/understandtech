Exam objective N10-009 2.2

Given a scenario, configure switching technologies and features

VLANs (Virtual Local Area Networks)

  • VLAN: Logically segments a physical network into separate broadcast domains.
  • Native VLAN: Untagged traffic on an 802.1Q trunk port. Default is VLAN 1 — change it for security.
  • Voice VLAN: A separate VLAN for VoIP traffic to ensure QoS prioritization.
  • 802.1Q Tagging: IEEE standard for VLAN tagging on trunk ports.
  • SVI (Switch Virtual Interface): A virtual interface on a Layer 3 switch used for inter-VLAN routing.

Spanning Tree Protocol (STP)

  • Purpose: Prevents Layer 2 loops in networks with redundant links by placing certain ports in a blocking state.
  • Root Bridge: The switch with the lowest Bridge ID (priority + MAC address) becomes the root bridge.
  • Port States: Blocking, Listening, Learning, Forwarding, Disabled.
  • RSTP (Rapid STP): Converges much faster than classic STP (seconds vs. minutes).

Exam tip

A symptom of a switching loop is a broadcast storm and extremely high CPU utilization on switches. STP is the solution. If STP is disabled and a loop exists, the network will flood with broadcast traffic until it collapses. Link Aggregation and MTU • LACP (Link Aggregation Control Protocol / 802.3ad): Combines multiple physical links into a single logical link for increased bandwidth and redundancy. • MTU (Maximum Transmission Unit): The largest packet size that can be transmitted. Standard Ethernet MTU = 1500 bytes. • Jumbo Frames: MTU greater than 1500 bytes (typically 9000 bytes). Used in storage networks (iSCSI, NFS) to reduce CPU overhead.
