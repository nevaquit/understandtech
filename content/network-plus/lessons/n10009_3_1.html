Explain the purpose of organizational processes and procedures

Documentation Types

Document Type Description

Physical Diagram Shows physical layout of devices, cables, and locations Logical Diagram Shows IP addressing, VLANs, routing protocols, and logical relationships Rack Diagram Shows equipment placement within server racks Cable Map Documents cable runs, labels, and termination points Network Diagram (L1/L2/L3) Layer-specific diagrams for troubleshooting

  • IPAM (IP Address Management): Tools and processes for planning, tracking, and managing IP address space to prevent conflicts.
  • SLA (Service-Level Agreement): A contract defining expected service levels, including uptime guarantees and response times.

Asset and Life-Cycle Management

  • EOL (End of Life): Vendor stops selling the product.
  • EOS (End of Support): Vendor stops providing updates and patches. Running EOS equipment is a major security risk.
  • Decommissioning: Securely removing and disposing of old equipment (data wiping, physical destruction).
  • Wireless Site Survey / Heat Map: A survey used to map RF coverage, identify dead zones, and locate sources of interference before or after AP deployment.

Change Management Process

A formal process for requesting, approving, and implementing changes to the network to minimize disruption.

  • Submit a Request for Change (RFC).
  • Assess impact and risk.
  • Obtain approval from the Change Advisory Board (CAB).
  • Schedule and implement the change.
  • Test and verify the change.
  • Document the outcome.

Configuration Management

  • Baseline/Golden Configuration: A known-good configuration used as a reference point.
  • Configuration Drift: When a device's running configuration deviates from the baseline over time.
